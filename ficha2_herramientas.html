Antes de programar necesitamos las herramientas correctas: el intérprete de Python y un buen editor de código. En esta clase instalamos todo y escribimos nuestras primeras líneas con la función print()

01¿Qué es un intérprete y por qué lo necesitamos?
ConceptoEl traductor entre tú y la computadora
Una computadora solo entiende código binario (ceros y unos). Cuando escribís print("Hola"), alguien tiene que traducir eso a algo que el procesador pueda ejecutar. Ese traductor es el intérprete de Python: un programa que lee tu código línea por línea, lo traduce y lo ejecuta al instante.
Sin el intérprete instalado, tu computadora no tiene forma de "entender" un archivo .py — sería solo texto sin sentido para ella.
Intérprete vs. Compilador¿Por qué Python usa un intérprete?
Algunos lenguajes (como C o C++) usan un compilador: traducen todo el programa de una sola vez antes de ejecutarlo. Python usa un intérprete: traduce y ejecuta línea por línea, en el momento.
VentajaPodés probar código al instante, sin esperar a compilar todo el programa — ideal para aprender y experimentar
CostoSuele ejecutarse un poco más lento que un programa ya compilado, aunque para la mayoría de tareas no se nota
02Instalando el intérprete — Python 3.13.13
Versión recomendada¿Por qué instalamos 3.13.13 y no la versión más nueva?
Python lanza versiones nuevas constantemente, y al momento de este curso la más reciente es más nueva que la 3.13.13. Sin embargo, vamos a instalar específicamente la 3.13.13 por una razón práctica: en el módulo final de este curso usamos Manim, una librería de animaciones, y las versiones de Python recién salidas suelen tardar un tiempo en tener soporte completo y estable en todos los sistemas e instaladores, incluyendo dependencias que usa Manim por debajo.
Regla práctica: cuando vayas a instalar una librería importante para un proyecto (como Manim), conviene usar una versión de Python ya probada y estable en vez de la más reciente del día. Así evitás errores de instalación difíciles de diagnosticar al principio del curso. Más adelante, con más experiencia, vas a poder evaluar tú mismo cuándo es seguro actualizar.
PasosCómo instalar Python 3.13.13
1
Entrar a python.org/downloads
En la sección de versiones anteriores (Releases), buscar específicamente Python 3.13.13 y descargar el instalador para tu sistema operativo.
2
Ejecutar el instalador
Al abrir el instalador en Windows, aparece abajo del todo una casilla muy importante: Add python.exe to PATH. Hay que marcarla siempre — si no, la consola no va a reconocer el comando python.
3
Verificar la instalación
Abrir una consola (CMD o PowerShell) y escribir el siguiente comando para confirmar que el intérprete responde correctamente.
python --version
# Salida esperada:
Python 3.13.13
03El editor de código — Visual Studio Code
Editor¿Qué es VS Code y por qué lo usamos?
VS Code (Visual Studio Code) es un editor de código gratuito, hecho por Microsoft, que se volvió el estándar de la industria. No es un lenguaje ni un compilador — es solo el "cuaderno" donde escribimos el código antes de que el intérprete lo ejecute.
Lo elegimos para este curso porque combina tres cosas difíciles de encontrar juntas: es liviano (no consume tantos recursos como otros editores grandes), tiene un ecosistema enorme de extensiones, y es gratuito y multiplataforma.
Comparativa¿Por qué no Sublime Text o Spyder?
EditorPor qué no lo elegimos para este curso
Sublime TextEs rápido y liviano, pero su soporte completo (autocompletado avanzado, debugging) requiere pagar una licencia o configurarlo a mano con más esfuerzo
SpyderEstá pensado especialmente para ciencia de datos (similar a RStudio o MATLAB) — es excelente para ese uso puntual, pero se siente pesado y poco flexible para programación general
Ninguno es "malo" — simplemente VS Code ofrece el mejor balance entre simplicidad inicial y crecimiento a futuro para alguien que recién empieza.
04Extensiones — el verdadero poder de VS Code
Extensiones¿Por qué importan tanto?
VS Code, recién instalado, es un editor de texto genérico — no sabe nada especial sobre Python. Las extensiones son pequeños complementos que se instalan con un clic y le agregan funciones específicas. Sin la extensión correcta, ni siquiera podrías ejecutar tu código de Python directamente desde el editor.
ExtensiónQué hace por nosotros
Python (Microsoft)La esencial: agrega resaltado de sintaxis, autocompletado, detección de errores y el botón "Run" para ejecutar el archivo
PylanceMejora el autocompletado y detecta errores de tipo antes de ejecutar el código, sugiriendo correcciones
Code RunnerPermite ejecutar fragmentos de código rápido con un botón, sin tener que usar siempre la terminal
05Tu primera función: print()
Funciónprint() — mostrar información en pantalla
print() es la función más usada de Python: muestra texto o valores en la consola. Acepta varios argumentos, no solo el texto a mostrar:
# Uso más simple
print("Hola, mundo")
# Hola, mundo

# Se pueden imprimir varios valores juntos
print("Edad:", 25, "años")
# Edad: 25 años
06El parámetro sep — separador entre valores
sep=""Por defecto, print() separa los valores con un espacio
Cuando imprimís varios valores separados por comas, Python los une con un espacio por defecto. Con sep podés cambiar ese separador por cualquier texto que quieras.
print("2026", "06", "19")
# 2026 06 19 (separador por defecto: espacio)

print("2026", "06", "19", sep="-")
# 2026-06-19 (ahora separados con guion)

print("manzana", "pera", "uva", sep=", ")
# manzana, pera, uva

print("sin espacio", "entre nada", sep="")
# sin espacioentre nada (sep vacío: todo pegado)
07El parámetro end — qué pasa al terminar de imprimir
end=""Por defecto, print() salta de línea al terminar
Cada print() termina agregando un salto de línea invisible (\n) — por eso cada print aparece en una línea nueva. Con end podés cambiar eso por cualquier otro texto, o incluso quitarlo.
print("Hola")
print("Mundo")
# Hola
# Mundo (cada print en su propia línea)

print("Hola", end=" ")
print("Mundo")
# Hola Mundo (quedaron en la misma línea)

print("Cargando", end="...")
print("listo")
# Cargando...listo
08Combinando sep y end — y otros parámetros que existen
Combinadosep y end trabajando juntos
Se pueden usar los dos parámetros en la misma línea para construir resultados más elaborados, como listas separadas por comas:
print("manzana", "pera", "uva", sep=", ", end=".\n")
# manzana, pera, uva.
Menos comunesfile y flush — los otros dos parámetros
Además de sep y end, print() acepta otros dos parámetros que verás con menos frecuencia, pero que existen y a veces resultan útiles:
file — Por defecto imprime en la consola, pero puede redirigir el texto hacia un archivo en lugar de mostrarlo en pantalla.
flush — Por defecto Python "amontona" la salida en un buffer antes de mostrarla. Con flush=True se fuerza a mostrarla de inmediato, útil en barras de progreso o cuentas en tiempo real.
09Tips para esta primera semana
sep y end son argumentos con nombre
Siempre se escriben como sep="..." y end="...", nunca solo el valor suelto. Python necesita saber a cuál de los dos te referís.
Verificá la instalación antes de seguir
Si python --version no responde en la consola, revisá que se haya marcado la casilla "Add to PATH" durante la instalación.
No instales decenas de extensiones
Con la extensión Python y Pylance alcanza para todo este curso. Instalar muchas extensiones desde el principio puede hacer más lento el editor.
end="" sin nada también es válido
end="" elimina por completo el salto de línea — útil cuando vas a seguir construyendo la misma línea con otro print más adelante.
Resumen
El intérprete traduce y ejecuta el código Python línea por línea, a diferencia de un compilador
Instalamos Python 3.13.13 por estabilidad y compatibilidad garantizada con Manim, no la versión más reciente
VS Code es liviano, gratuito y tiene un ecosistema enorme de extensiones
Las extensiones le agregan a VS Code capacidades que no tiene de fábrica, como ejecutar Python
sep cambia el separador entre valores · end cambia qué se imprime al finalizar (por defecto, salto de línea)
file y flush son parámetros menos comunes de print(), para redirigir salida o forzar que se muestre al instante
