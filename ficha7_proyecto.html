02tkinter vs ttk — ¿cuál usar?
tkinterWidgets básicos
Los widgets de tkinter son simples y funcionales. Permiten personalizar colores y fuentes fácilmente con bg y fg. Se usan para Label, Button, Entry, Frame.
import tkinter as tk

# Control total de colores
tk.Label(ventana,
    text="Título",
    bg="#ffffff",
    fg="#1e293b",
    font=("Segoe UI", 20, "bold")
)
ttkWidgets con estilo nativo
ttk (Themed Tk) ofrece widgets de aspecto más moderno y nativo del sistema operativo. Incluye Treeview, Style, Combobox y más. Los colores se controlan con Style, no con bg/fg.
from tkinter import ttk

# Treeview — tabla avanzada
tabla = ttk.Treeview(frame)

# Estilo personalizado
estilo = ttk.Style()
estilo.configure("Treeview",
    rowheight=30)
03Frame — organizar la interfaz en secciones
Un Frame es un contenedor invisible que agrupa widgets. Permite organizar la ventana en zonas independientes — cada zona se posiciona y estila por separado.
Los 3 frames del proyecto
# Frame 1: encabezado blanco
frame_header = tk.Frame(
    ventana, bg="#ffffff", pady=15
)
frame_header.pack(fill=tk.X) # ancho total

# Frame 2: zona de botones
frame_controles = tk.Frame(
    ventana, bg="#f3f4f6"
)
frame_controles.pack(pady=20)

# Frame 3: zona de tabla
frame_tabla = tk.Frame(
    ventana, bg="#f3f4f6"
)
frame_tabla.pack(
    fill=tk.BOTH, expand=True,
    padx=30, pady=(0, 20)
)
fill y expand — ¿qué hacen?
fill=tk.X — el frame ocupa todo el ancho disponible. Usado en el header para que se extienda de borde a borde.
fill=tk.BOTH — el frame ocupa todo el ancho y alto disponible. Se usa en la zona de la tabla.
expand=True — permite que el frame crezca si la ventana se agranda. Sin esto, la tabla no se estira al redimensionar.
pady=(0, 20) — tupla que define margen arriba=0, abajo=20. Más preciso que un solo número.

05Treeview — tabla de datos
TreeviewCrear y configurar la tabla
Treeview es el widget de tabla más completo de tkinter. Se configura en 3 pasos: definir columnas, poner encabezados y ajustar el ancho de cada columna.
# 1. Crear con columnas
tabla = ttk.Treeview(
    frame_tabla,
    columns=("ID", "Producto", "Precio"),
    show="headings" # oculta col vacía
)

# 2. Definir encabezados
tabla.heading("ID", text="ID")
tabla.heading("Producto", text="Nombre del Producto")
tabla.heading("Precio", text="Precio (USD)")

# 3. Ajustar columnas
tabla.column("ID", width=50, anchor=tk.CENTER)
tabla.column("Producto", width=400, anchor=tk.W)
tabla.column("Precio", width=150, anchor=tk.E)
tabla.pack(fill=tk.BOTH, expand=True)
insert / deleteInsertar y limpiar filas
Para cargar datos se usa .insert(). Antes de cargar nuevos datos siempre hay que limpiar la tabla con .delete() para evitar duplicados.
def descargar_dato():
    # 1. Limpiar tabla antes de cargar
    for fila in tabla.get_children():
        tabla.delete(fila)

    # 2. Datos de prueba (lista de tuplas)
    datos = [(1, "Monitor 24'", 150.50),
              (2, "Teclado RGB", 45.00)]

    # 3. Insertar cada fila
    for item in datos:
        tabla.insert("", tk.END,
            values=(
                item[0], item[1],
                f"${item[2]:.2f}" # formato $
            )
        )
06ttk.Style — personalizar apariencia de widgets ttk
Los widgets de ttk no aceptan bg ni fg directamente. Su apariencia se controla con ttk.Style, que permite cambiar fuente, alto de fila, color de fondo y más.
estilo = ttk.Style()

# Personalizar encabezados
estilo.configure(
    "Treeview.Heading",
    font=("Segoe UI", 10, "bold")
)

# Personalizar filas de la tabla
estilo.configure(
    "Treeview",
    rowheight=30 # alto de cada fila
)
"Treeview.Heading" — controla el estilo de la fila de encabezados (ID, Producto, Precio)
"Treeview" — controla el estilo de las filas de datos: alto, fuente, colores
rowheight=30 — aumenta la altura de cada fila para que el contenido se lea con más comodidad
⚠ Style se aplica ANTES de crear el Treeview para que los cambios tomen efecto correctamente

import tkinter as tk
from tkinter import messagebox
from tkinter import ttk

# ── FUNCIONES ──────────────────────────────
def descargar_dato():
    # Limpiar filas existentes
    for fila in tabla.get_children():
        tabla.delete(fila)
    # Datos de prueba (simulan una API)
    datos_prueba = [
        (1, "Monitor Curvo 24'", 150.50),
        (2, "Teclado Mecánico RGB", 45.00),
        (3, "Silla Ergonómica Ejecutiva", 120.99)
    ]
    for item in datos_prueba:
        tabla.insert("", tk.END, values=(item[0], item[1], f"${item[2]:.2f}"))

def exportar_excel():
    messagebox.showinfo("Próximamente", "Aquí se hará el excel")

def mostrar_graficos():
    messagebox.showinfo("Próximamente", "Aquí se verán los gráficos")

# ── VENTANA ────────────────────────────────
ventana = tk.Tk()
ventana.title("Dashboard Analítico")
ventana.geometry("900x550")
ventana.configure(bg="#f3f4f6")

tk.Label(frame_header, text="Módulo 1: Interfaz Visual", font=("Segoe UI", 11), bg="#ffffff", fg="#64748b").pack()

frame_controles = tk.Frame(ventana, bg="#f3f4f6")
frame_controles.pack(pady=20)
tk.Button(frame_controles, text="☁️ Descargar (API)", command=descargar_dato, bg="#0284c7", fg="white", font=("Segoe UI", 10, "bold"), width=20).grid(row=0, column=0, padx=10)

frame_tabla.pack(fill=tk.BOTH, expand=True, padx=30, pady=(0, 20))

08Tips y Buenas Prácticas
Funciones antes de la ventana
Define siempre las funciones al inicio, antes de crear widgets. Si el botón llama a descargar_dato y aún no está definida, Python lanza un error.
Limpiar antes de insertar
Siempre usa tabla.get_children() + tabla.delete() antes de cargar nuevos datos. Sin esto los datos anteriores se acumulan con cada clic.
f-string para formatear precio
f"${item[2]:.2f}" formatea el número con exactamente 2 decimales. Sin esto, 45.0 se mostraría como 45.0 en vez de $45.00.
messagebox como marcador
Usar messagebox.showinfo("Próximamente", "...") en funciones no implementadas es una buena práctica — la app funciona sin errores mientras desarrollas por partes.
Resumen
ttk extiende tkinter con widgets más modernos como Treeview y Style
Frame organiza la ventana en zonas independientes y manejables
fill=X ocupa el ancho · fill=BOTH ocupa ancho y alto · expand=True crece
Treeview necesita: columns → heading → column → insert para funcionar
ttk.Style controla apariencia de widgets ttk (no bg/fg directamente)
messagebox permite simular funciones pendientes sin romper la app
