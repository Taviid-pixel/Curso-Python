01 · APIs
¿Qué es?API — Application Programming Interface
API¿Qué es una API?
Una API es un puente entre tu programa y un servidor en internet. Le haces una pregunta (request) y el servidor te responde con datos (response). Es como pedir el menú en un restaurante — tú pides, el cocinero prepara y el mesero te trae la respuesta.
GET → Tu programa envía una petición a una URL
200 OK ← El servidor responde con datos en formato JSON
ERROR ← Si algo falla: sin internet, URL incorrecta, servidor caído
¿Para qué sirve?Casos de uso reales
Las APIs permiten obtener datos en tiempo real sin necesidad de tener archivos propios. Todo lo que ves en apps conectadas a internet usa APIs:
☁️ Clima — obtener temperatura y pronóstico actuales
🛒 Catálogos — productos, precios e inventario actualizados
💱 Divisas — tipo de cambio en tiempo real
📰 Noticias — titulares actualizados cada minuto
Libreríarequests — cómo instalar y usar
pip installInstalar la librería requests
Python no incluye requests por defecto — hay que instalarla. Se instala una sola vez desde la terminal usando pip, el gestor de paquetes de Python.
# Abrir terminal / símbolo del sistema y ejecutar:
pip install requests

# Verificar que se instaló correctamente:
pip show requests
⚠ pip (Pip Installs Packages) descarga la librería desde internet y la guarda en tu Python. Solo necesitas hacerlo una vez por computadora.
requests.get()Hacer una petición GET
El método GET es el más común — pide datos al servidor. La respuesta tiene un código de estado y los datos en JSON.
import requests

# Hacer la petición
respuesta = requests.get("https://fakestoreapi.com/products")

# Ver el código de estado (200 = éxito)
print(respuesta.status_code) # 200

# Convertir respuesta a datos Python
datos = respuesta.json() # lista de dicts
print(datos[0]) # primer producto
Funcióndescargar_datos() — función completa del proyecto
Esta función conecta con fakestoreapi.com, descarga el catálogo de productos y lo muestra en la tabla del Dashboard. Usa try/except para manejar errores de conexión sin que el programa se cierre:

        # Limpiar tabla

        # Insertar cada producto

Paso a paso
1. requests.get(url) — envía la petición HTTP al servidor de fakestoreapi
2. respuesta.json() — convierte el texto JSON de la respuesta en una lista de diccionarios Python
3. item["id"], item["title"], item["price"] — accede a cada campo del diccionario por su clave
4. try/except — si no hay internet o falla la conexión, captura el error y muestra un mensaje amigable en vez de romper el programa
02 · Excel
¿Qué es?pandas — leer archivos Excel con Python
pandas¿Qué es pandas?
pandas es la librería más popular de Python para manejar datos en tabla. Permite leer archivos Excel, CSV, filtrar filas, calcular estadísticas y mucho más. Los datos se almacenan en un DataFrame — como una hoja de Excel dentro de Python.
# Instalar pandas y openpyxl (para leer .xlsx)
pip install pandas openpyxl

# Importar en tu código
import pandas as pd

# Leer un archivo Excel
df = pd.read_excel("archivo.xlsx")
print(df) # muestra la tabla completa
DataFrame¿Qué es un DataFrame?
Un DataFrame es una tabla de datos con filas y columnas, igual que Excel. Cada columna tiene un nombre y cada fila un índice. Para recorrer las filas se usa iterrows().
df = pd.read_excel("Reporte_Mercado.xlsx")

# Acceder a una columna completa
print(df["Producto"])

# Recorrer fila por fila
for index, row in df.iterrows():
    print(row["ID"], row["Producto"])

# index = número de fila (0,1,2...)
# row = datos de esa fila como dict
Funcióndescargar_datos() — cargar desde Excel
Esta función lee el archivo Reporte_Mercado.xlsx y carga los datos en la tabla del Dashboard. Incluye conversión del precio que viene con el texto "Bs":
import pandas as pd

def descargar_datos():
    try:
        df = pd.read_excel("Reporte_Mercado.xlsx")

        # Limpiar tabla
        for fila in tabla.get_children():
            tabla.delete(fila)

        # Recorrer el DataFrame fila por fila
        for index, row in df.iterrows():
            id_prod = row["ID"]
            nombre = row["Producto"]
            # Limpiar "Bs" del precio y convertir a float
            precio = float(
                str(row["Precio"])
                .replace("Bs", "").strip()
            )
            tabla.insert("", tk.END,
                values=(id_prod, nombre, f"Bs{precio:.2f}"))

        messagebox.showinfo("Éxito", "Datos cargados desde Excel")
    except Exception as e:
        messagebox.showerror("Error", str(e))
Puntos clave
pd.read_excel() — lee el archivo .xlsx y lo convierte en un DataFrame. El archivo debe estar en la misma carpeta que el script.
df.iterrows() — devuelve cada fila como un par (índice, datos). Se usa en un for para recorrer la tabla fila por fila.
str().replace("Bs","").strip() — el precio puede venir como "Bs15.00". Se elimina el texto "Bs" y los espacios, luego se convierte a float para operar con él.
except Exception as e — captura cualquier error y lo muestra con str(e). Útil para ver exactamente qué falló.
03 · JSON
¿Qué es?JSON — formato de intercambio de datos
JSON¿Qué es JSON?
JSON (JavaScript Object Notation) es un formato de texto para guardar y transportar datos. Es legible por humanos y fácil de procesar por cualquier lenguaje de programación. Python lo maneja con la librería json incluida — no necesita instalación.
[
  {
    "id": 1,
    "title": "Arroz 1kg",
    "price": 250.0
  },
  {
    "id": 2,
    "title": "Azúcar 1kg",
    "price": 7.8
  }
]
EstructuraJSON en Python
JSON se convierte automáticamente en tipos Python: un objeto {} → diccionario, un array [] → lista, strings → str, números → int/float, true/false → True/False.
import json

# LEER un archivo JSON
with open("productos.json", "r",
          encoding="utf-8") as archivo:
    datos = json.load(archivo) # lista

# GUARDAR un JSON
with open("productos.json", "w",
          encoding="utf-8") as archivo:
    json.dump(datos, archivo,
            indent=4, ensure_ascii=False)
PathBASE_DIR — encontrar el archivo en cualquier computadora
El problema clásico: en tu computadora el archivo está en C:\Users\Taviid\proyecto\ pero en otra computadora el path es diferente. La solución es calcular la ruta relativa al propio script:
Código completo del path
import os
import sys

# ¿El programa corre como .exe o como .py?
if getattr(sys, "frozen", False):
    # Corriendo como .exe (PyInstaller)
    BASE_DIR = os.path.dirname(sys.executable)
else:
    # Corriendo como script .py normal
    BASE_DIR = os.path.dirname(
        os.path.abspath(__file__)
    )

# Construir la ruta completa al JSON
RUTA_JSON = os.path.join(BASE_DIR, "productos_minimarket.json")
¿Por qué es necesario?
sys.frozen — cuando empaquetas el programa con PyInstaller como .exe, __file__ ya no existe. sys.frozen detecta si está corriendo como ejecutable.
os.path.abspath(__file__) — obtiene la ruta absoluta del script .py actual, sin importar desde dónde se ejecute.
os.path.join() — une la carpeta base con el nombre del archivo de forma correcta en cualquier sistema operativo (Windows, Mac, Linux).

json.dumpguardar_json() — escribir JSON
Lee los datos actuales de la tabla y los guarda de vuelta en el archivo JSON. Convierte el precio limpiando el texto "Bs" antes de guardarlo como número.
def guardar_json():
    datos = []

    # Leer cada fila de la tabla
    for fila in tabla.get_children():
        valores = tabla.item(fila)["values"]
        datos.append({
            "id": int(valores[0]),
            "title": valores[1],
            "price": float(
                str(valores[2])
                .replace("Bs", "").strip()
            )
        })

    # Guardar en archivo
    with open(RUTA_JSON, "w",
            encoding="utf-8") as archivo:
        json.dump(datos, archivo,
                indent=4, ensure_ascii=False)

TipsBuenas Prácticas
Siempre usar try/except
Toda operación con datos externos (API, archivo) puede fallar. Envuélvela en try/except para mostrar un mensaje útil al usuario en vez de un error técnico.
encoding="utf-8" siempre
Al abrir archivos JSON con open(), siempre especifica encoding="utf-8". Sin esto, los acentos (á, é, ñ) pueden causar errores en Windows.
ensure_ascii=False en json.dump
Al guardar JSON con json.dump(), usa ensure_ascii=False para que los caracteres especiales (ñ, acentos) se guarden correctamente y no como códigos \u00f1.
BASE_DIR para paths portables
Nunca escribas rutas absolutas como C:\Users\.... Usa siempre BASE_DIR + os.path.join() para que el programa funcione en cualquier computadora.
