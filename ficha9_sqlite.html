01¿Qué es una base de datos?
Base de datos¿Qué es y para qué sirve?
Una base de datos es un sistema organizado para guardar información de forma permanente. A diferencia de variables que desaparecen al cerrar el programa, una base de datos persiste en un archivo y permite buscar, filtrar y modificar datos de forma eficiente.
🗄️ Persistencia — los datos se conservan aunque el programa se cierre
⚡ Eficiencia — modifica solo lo que cambia, no reescribe todo
🔍 Consultas — buscar y filtrar datos con SQL en milisegundos
🔒 Integridad — controla qué datos se pueden guardar con tipos y restricciones
SQLite3¿Por qué SQLite?
SQLite es una base de datos que guarda todo en un único archivo .db en tu computadora. No necesita servidor, no requiere instalación y viene incluida en Python. Perfecta para aplicaciones de escritorio como nuestro proyecto.
# Ya viene con Python — no hay que instalar nada
import sqlite3

# Crea el archivo minimarket.db si no existe
con = sqlite3.connect("minimarket.db")
✅ Sin instalación · Sin servidor · Un solo archivo · Incluido en Python

03RUTA_DB y obtener_conexion() — setup inicial
RUTA_DBDel .json al .db
El mismo patrón BASE_DIR + os.path.join() del módulo anterior, pero ahora apunta a un archivo .db en vez de .json. Si el archivo no existe, SQLite lo crea automáticamente.
# Antes: apuntaba al JSON
RUTA_JSON = os.path.join(BASE_DIR, "productos.json")

# Ahora: apunta a la base de datos
RUTA_DB = os.path.join(BASE_DIR, "minimarket.db")

# SQLite crea el .db solo si no existe
# No necesitas crearlo manualmente
obtener_conexion()Función de conexión
Centraliza la apertura de la conexión en una sola función. Todas las funciones que necesiten leer o escribir en la BD la llaman. Así si cambia la ruta, solo se modifica en un lugar.
def obtener_conexion():
    return sqlite3.connect(RUTA_DB)

# Se usa con 'with' para cerrar automático
with obtener_conexion() as con:
    con.execute("...")
    con.commit()
# Al salir del 'with', cierra solo
04inicializar_bd() — crear la tabla al arrancar
Esta función se ejecuta una sola vez al arrancar el programa desde main(). Crea la tabla productos si no existe. Si ya existe, no hace nada — no borra datos.
def inicializar_bd():
    with obtener_conexion() as con:
        con.execute("""
            CREATE TABLE IF NOT EXISTS productos (
                id INTEGER PRIMARY KEY AUTOINCREMENT,
                title TEXT NOT NULL,
                price REAL NOT NULL
            )
        """)
        con.commit()

# Se llama al inicio del programa
def main():
    inicializar_bd() # primero la BD
    ventana = construir_ui()
    ventana.mainloop()
Cada parte de CREATE TABLE
1
CREATE TABLE IF NOT EXISTS
Crea la tabla solo si no existe. Si ya existe (segunda ejecución), no hace nada y no borra los datos existentes.
2
INTEGER PRIMARY KEY AUTOINCREMENT
INTEGER = número entero · PRIMARY KEY = identificador único · AUTOINCREMENT = SQLite asigna el ID solo, reemplaza a generar_id()
3
TEXT NOT NULL
El título es texto y obligatorio. SQLite rechaza cualquier INSERT sin este campo.
4
REAL NOT NULL
REAL = número decimal (equivale a float en Python). El precio es obligatorio y debe ser un número.
5
con.commit()
Guarda los cambios permanentemente en el archivo .db. Sin commit(), los cambios se pierden al cerrar la conexión.
05CRUD — las 4 operaciones de toda base de datos
CRUD = Create · Read · Update · Delete. Son las 4 operaciones básicas de cualquier base de datos. En el proyecto cada una tiene su propia función y su propio comando SQL.
C
db_insertar() — CREATE
INSERT INTO productos
Agrega un producto nuevo. SQLite asigna el ID automáticamente y lo retorna con lastrowid para mostrarlo en la tabla visual.
def db_insertar(nombre, precio):
  with obtener_conexion() as con:
    cursor = con.execute(
      "INSERT INTO productos (title, price) VALUES (?, ?)",
      (nombre, precio)
    )
    con.commit()
    return cursor.lastrowid # ID real
? son marcadores de posición — SQLite inserta los valores de la tupla en orden. Esto protege contra inyección SQL.
R
db_obtener_todos() — READ
SELECT * FROM productos
Lee todos los productos de la BD ordenados por ID. Retorna una lista de tuplas: [(1,'Arroz',12.5), (2,'Azúcar',8.0)]
def db_obtener_todos():
  with obtener_conexion() as con:
    cursor = con.execute(
      "SELECT id, title, price FROM productos ORDER BY id"
    )
    return cursor.fetchall()
fetchall() retorna todas las filas · ORDER BY id las ordena de menor a mayor.
U
db_actualizar() — UPDATE
UPDATE productos SET ... WHERE id=?
Modifica el nombre y precio de solo el producto con ese ID. No toca el resto. Antes esto requería reescribir todo el JSON.
def db_actualizar(prod_id, nombre, precio):
  with obtener_conexion() as con:
    con.execute(
      "UPDATE productos SET title=?, price=? WHERE id=?",
      (nombre, precio, prod_id)
    )
    con.commit()
WHERE id=? es clave — sin él, actualizaría todos los productos.
D
db_eliminar() — DELETE
DELETE FROM productos WHERE id=?
Elimina el producto con ese ID de la BD. En el proyecto, primero se borra de la BD y luego se borra de la tabla visual.
def db_eliminar(prod_id):
  with obtener_conexion() as con:
    con.execute(
      "DELETE FROM productos WHERE id=?",
      (prod_id,) # tupla de 1 elemento
    )
    con.commit()
(prod_id,) — la coma hace que sea una tupla de 1 elemento. Sin coma, Python lo trata como un paréntesis normal, no como tupla.

07SQL — los comandos usados en el proyecto
SQL¿Qué es SQL?
SQL (Structured Query Language) es el lenguaje para hablar con bases de datos. Es el mismo en SQLite, MySQL, PostgreSQL y otros. Una vez que aprendes SQL básico, funciona en cualquier base de datos.
── Comandos usados en el proyecto ──

CREATE TABLE IF NOT EXISTS tabla (...)
→ Crea la tabla si no existe

SELECT col1, col2 FROM tabla ORDER BY id
→ Lee y ordena registros

INSERT INTO tabla (col1, col2) VALUES (?,?)
→ Inserta un registro nuevo

UPDATE tabla SET col1=? WHERE id=?
→ Modifica solo la fila que coincide

DELETE FROM tabla WHERE id=?
→ Borra solo la fila que coincide
Tipos SQLiteTipos de datos en SQLite
SQLite usa tipos de datos propios que se mapean a tipos Python automáticamente al leer con fetchall():

NOT NULL — garantiza que el campo no puede estar vacío. SQLite lanzará un error si intentas insertar sin ese campo.
08Tips y Buenas Prácticas
Siempre usar ? en vez de f-string
Nunca construyas SQL con f-strings: f"WHERE id={id}". Usa siempre ? con tuplas. Los f-strings abren la puerta a inyección SQL — un ataque que destruye la base de datos.
commit() después de escribir
commit() guarda los cambios permanentemente. Sin él, INSERT, UPDATE y DELETE son temporales y se pierden al cerrar la conexión. SELECT no necesita commit.
with → cierre automático
Usar with obtener_conexion() as con: cierra la conexión automáticamente al salir del bloque. Evita conexiones abiertas que consumen memoria.
(prod_id,) — tupla de un elemento
Cuando el SQL tiene solo un ?, el segundo argumento debe ser una tupla: (prod_id,) con coma al final. Sin la coma, Python lo trata como número, no como tupla, y SQLite lanza un error.
