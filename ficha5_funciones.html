01¿Qué es una función y por qué usarla?
defDefinición de función
Una función es un bloque de código con nombre que se puede ejecutar cuantas veces quieras. Se define con def y se llama escribiendo su nombre seguido de paréntesis.
# Definir la función
def saludar():
    print("¡Hola desde la función!")

# Llamar la función
saludar() # ¡Hola desde la función!
saludar() # se puede llamar N veces
¿Por qué?Ventajas de usar funciones
Las funciones resuelven un problema clave: no repetir código. Si algo cambia, lo modificas en un solo lugar.
♻️ Reutilización — escribe una vez, llama las veces que necesites
🧩 Organización — divide el programa en partes pequeñas y claras
🐛 Fácil de depurar — si hay un error, sabes exactamente dónde buscar
02Estructura de una función
Toda función en Python sigue esta estructura. Cada parte tiene un rol específico:
def calcular_area(base, altura):
    """Calcula el área de un triángulo"""
    area = (base * altura) / 2
    return area

# Llamada con argumentos
resultado = calcular_area(10, 5)
print(resultado) # 25.0
def Palabra reservada que indica el inicio de una función
nombre Identificador único. Usa snake_case: calcular_area
( ) Parámetros que recibe la función. Pueden ser cero o más
body Cuerpo con 4 espacios de indentación. Aquí va la lógica
return Devuelve un valor al llamador. Opcional — sin return devuelve None
03Parámetros y Argumentos
Parámetro = variable en la definición  ·  Argumento = valor que se pasa al llamar. Python soporta varios tipos:
TipoDefiniciónLlamadaDescripción
Posicional def f(a, b): f(1, 2) Se asignan por orden de posición. El más común.
Por defecto def f(a, b=10): f(5)  → b=10 Si no se pasa el argumento, usa el valor por defecto.
Nombrado def f(a, b): f(b=2, a=1) Se indica el nombre del parámetro. El orden no importa.
*args def f(*nums): f(1,2,3,4) Recibe cantidad variable de argumentos como tupla.
Posicional + Defecto
El parámetro con valor por defecto siempre va al final de la lista.
def saludar(nombre, saludo="Hola"):
    print(f"{saludo}, {nombre}!")

saludar("Ana") # Hola, Ana!
saludar("Luis", "Buenos días")
*argsCantidad variable
Útil cuando no sabes cuántos argumentos recibirás. Python los agrupa en una tupla.
def sumar(*numeros):
    total = 0
    for n in numeros:
        total += n
    return total

sumar(1, 2) # 3
sumar(1, 2, 3, 4) # 10
04Retorno de Valores — return
returnDevolver un valor
return termina la función y devuelve un valor al código que la llamó. Ese valor puede guardarse en una variable o usarse directamente.
def cuadrado(n):
    return n ** 2

resultado = cuadrado(5)
print(resultado) # 25
print(cuadrado(3)) # 9
return múltipleDevolver varios valores
Python permite devolver más de un valor separado por comas. Se reciben como tupla o desempaquetados.
def min_max(lista):
    return min(lista), max(lista)

nums = [3, 1, 7, 2]
minimo, maximo = min_max(nums)
print(minimo, maximo) # 1 7
05Alcance de Variables — Scope
El scope define dónde es visible una variable. Las variables creadas dentro de una función son locales — no existen fuera de ella.
Variable LOCAL
Creada dentro de la función. Solo existe dentro de ella. Al terminar la función, desaparece.
def mi_funcion():
    x = 10 # local
    print(x) # ✓ funciona

mi_funcion()
# print(x) → NameError ✗
Variable GLOBAL
Creada fuera de toda función. Visible en todo el programa. Se puede leer dentro de funciones, pero para modificarla se usa global.
mensaje = "Hola" # global

def mostrar():
    print(mensaje) # ✓ lee global

def cambiar():
    global mensaje
    mensaje = "Chau" # modifica
06Funciones Lambda — funciones anónimas
lambda¿Qué es una lambda?
Una función sin nombre definida en una sola línea. Útil para operaciones simples y rápidas. Sintaxis: lambda parámetros: expresión
# Función normal
def doble(x):
    return x * 2

# Equivalente con lambda
doble = lambda x: x * 2

print(doble(5)) # 10
lambda + sortedUso práctico
Las lambdas brillan cuando se combinan con funciones como sorted(), map() o filter().
alumnos = [
  {"nombre": "Ana", "nota": 90},
  {"nombre": "Luis", "nota": 75}
]
# Ordenar por nota
ordenados = sorted(alumnos,
    key=lambda a: a["nota"])
07Ejercicio integrador
Este ejercicio combina funciones, parámetros, return múltiple y desempaquetado en un solo programa real:
Código completo
def analizar(lista):
    mayor = max(lista)
    menor = min(lista)
    promedio = sum(lista) / len(lista)
    return mayor, menor, promedio

numeros = [12, 45, 23, 67, 34]
m, n, p = analizar(numeros)

print("mayor", m)
print("menor", n)
print("promedio", p)
¿Qué conceptos aplica?
def analizar(lista) → recibe una lista como parámetro posicional
max(), min(), sum(), len() → funciones integradas de Python usadas dentro de nuestra función
return mayor, menor, promedio → devuelve 3 valores a la vez como tupla
m, n, p = analizar(...) → desempaquetado de la tupla retornada en 3 variables
Salida esperada:
mayor    67
menor    12
promedio 36.2
08Tips y Buenas Prácticas
Una función, una tarea
Cada función debe hacer una sola cosa y hacerla bien. Si el nombre necesita "y", probablemente son dos funciones.
Docstring
Agrega """descripción""" justo después del def. Documenta qué hace, qué recibe y qué devuelve.
Evitar variables globales
Usa return para pasar datos entre funciones en lugar de depender de variables globales. El código será más limpio y predecible.
Nombres descriptivos
Usa verbos: calcular_precio(), obtener_usuario(). Un buen nombre hace que el código se lea como inglés o español natural.
Resumen
def nombra y define la función — no la ejecuta, solo la prepara
Los parámetros son variables de entrada; los argumentos son los valores reales
return devuelve un valor — sin él la función devuelve None
Variables locales solo existen dentro de la función donde se crean
lambda es una función anónima de una sola línea para operaciones simples
Una función debe hacer una sola cosa y tener un nombre descriptivo
