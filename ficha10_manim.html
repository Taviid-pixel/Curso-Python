Manim — Animando lo que aprendiste
Manim Animaciones con código
Cerramos el curso con algo distinto: usar todo lo que aprendiste (clases, objetos, métodos) para crear animaciones matemáticas y visuales con Python. Manim es la librería que usa 3Blue1Brown para sus videos — ahora la vas a usar tú
manim Scene class construct() self.play() Transform
01¿Qué es Manim y para qué sirve?
LibreríaManim — Mathematical Animation Engine
Manim es una librería de Python para crear animaciones matemáticas y educativas mediante código. Fue creada por Grant Sanderson, el creador del canal de YouTube 3Blue1Brown, para producir sus videos de matemáticas. Hoy existe una versión mantenida por la comunidad llamada Manim Community (ManimCE), que es la que vamos a usar — más estable, mejor documentada y más fácil de instalar.
# Instalación
pip install manim
Uso real¿Dónde se usa Manim?
No es solo para matemáticas. Se usa para:
📐 Educación — explicar conceptos abstractos (física, álgebra, geometría) de forma visual
🎥 Contenido para YouTube/redes — canales educativos como 3Blue1Brown
💻 Visualizar código — mostrar cómo cambian estructuras de datos, algoritmos, lógica
📊 Presentaciones técnicas — animar diagramas en charlas o clases
02try.manim.community — probarlo sin instalar nada
Sandbox online¿Qué es try.manim.community?
Es un cuaderno Jupyter en línea que corre directo en el navegador, sin que tengas que instalar Python ni Manim en tu computadora. Es la forma más rápida de probar código de Manim antes de animarte a instalarlo localmente — ideal para los primeros experimentos o para mostrarle algo a alguien sin pedirle que instale nada.
Importante: en try.manim.community la librería from manim import * ya viene ejecutada en una celda inicial del cuaderno. Vos no la volvés a escribir — solo necesitas otra celda nueva con tu class y, justo arriba de ella, la línea mágica %%manim que le dice al cuaderno "esto es Manim, renderízalo". Si después instalas Manim en tu propia computadora (con pip install manim), ahí sí necesitas escribir el from manim import * al principio de tu archivo .py — try.manim.community es la única excepción porque ya te lo deja puesto.
03Antes de animar: ¿qué es una clase?
POOProgramación Orientada a Objetos (POO)
Hasta ahora programaste con funciones: bloques de código que reciben datos y devuelven un resultado. Una clase es distinto: es un molde para crear objetos que tienen tanto datos (atributos) como comportamientos (métodos) juntos en un solo paquete. Manim está construido sobre clases — por eso toda animación empieza heredando de Scene.
classDefine una clase nueva. MiPrimeraAnimacion(Scene) significa "esta clase hereda todo el comportamiento de Scene" — ya viene con play(), wait(), add(), etc.
constructEs el método que Manim ejecuta automáticamente para renderizar la escena. Todo lo que pasa en pantalla se escribe adentro de este método.
selfHace referencia "a esta misma escena". self.play(...) significa "reproduce esta animación en MI escena".
métodoEs una función que vive dentro de una clase. play() y wait() son métodos heredados de Scene, listos para usar.
04Tu primera animación, línea por línea
En try.manim.communityCelda nueva — la librería ya está importada
%%manim -qm -v WARNING MiPrimeraAnimacion

class MiPrimeraAnimacion(Scene):
  def construct(self):
    texto = Text("Felicidades por terminar el curso", font_size=36)
    cuadrado = Square(color=BLUE, fill_opacity=0.5)
    circulo = Circle(color=RED, fill_opacity=0.8)

    texto.to_edge(UP)
    cuadrado.shift(LEFT * 2)
    circulo.shift(RIGHT * 2)

    self.play(Write(texto))
    self.wait(5)

    self.play(Create(cuadrado))
    self.wait(5)

    self.play(Create(circulo))
    self.wait(5)

    self.play(Transform(cuadrado, circulo))
    self.wait(5)
%%manim -qm -v WARNING MiPrimeraAnimacion es la línea mágica de Jupyter que renderiza la escena. -qm pide calidad media y -v WARNING reduce los mensajes de la consola, mostrando solo advertencias importantes.
En tu computadora (.py)mi_animacion.py — necesita el import
Si guardas este mismo código en un archivo de tu computadora, ahí sí tenés que agregar la línea de importación al principio, porque tu archivo no tiene ninguna celda previa que ya la haya cargado:
from manim import *

class MiPrimeraAnimacion(Scene):
  ... (el resto del código es igual)
05Desglose: qué hace cada bloque
Flujo de la escenaEl orden en que ocurre todo
1
Se crean los objetos
Text(), Square() y Circle() son objetos — instancias de clases que ya vienen dentro de Manim. Cada uno guarda sus propios datos: color, tamaño, posición.
2
Se posicionan antes de mostrarse
to_edge(UP) manda el texto arriba. shift(LEFT * 2) y shift(RIGHT * 2) separan el cuadrado y el círculo para que no queden encimados.
3
self.play() dispara la animación
Write() dibuja el texto como si se escribiera a mano. Create() dibuja la figura trazando su contorno. Sin play(), el objeto existiría pero nunca aparecería en pantalla.
4
self.wait() pausa la escena
Sin esto, Manim pasaría a la siguiente animación de inmediato. wait(5) deja la imagen 5 segundos en pantalla para que se pueda apreciar.
5
Transform() morfa un objeto en otro
Transform(cuadrado, circulo) anima la transición del cuadrado hasta convertirse visualmente en el círculo — una de las animaciones más icónicas de Manim.
06Cómo ejecutar tu animación
OnlineOpción 1 — try.manim.community
Escribís tu class en una celda nueva, le ponés arriba la línea mágica %%manim -qm -v WARNING MiPrimeraAnimacion y ejecutás la celda. No hace falta repetir el from manim import * — ya está cargado.
TerminalOpción 2 — en tu computadora
Guardás el código (con su from manim import *) en un archivo, por ejemplo mi_animacion.py, y desde la consola — no presionas "Play" como en otros programas — ejecutás:
manim -pql mi_animacion.py MiPrimeraAnimacion
Parte del comando Qué significa
manimEl programa que renderiza la animación
-pPreview — abre el video automáticamente al terminar de renderizar
-qlQuality low — renderiza rápido en baja calidad (bueno para probar)
mi_animacion.pyEl archivo donde guardaste la clase
MiPrimeraAnimacionEl nombre exacto de la clase que querés renderizar — debe coincidir con el class del código
Más flagsNo solo -ql: las 5 calidades de renderizado
El flag de calidad siempre empieza con -q seguido de una letra. Mientras más alta la calidad, más tarda en renderizar:
Flag Calidad Resolución
-qlBaja (Low)480p · 15 fps
-qmMedia (Medium)720p · 30 fps
-qhAlta (High)1080p · 60 fps
-qp2K1440p · 60 fps
-qk4K2160p · 60 fps
Hay más flags además de calidad y preview: -f abre la carpeta donde quedó el video en vez de reproducirlo · -i exporta como .gif en vez de .mp4 · -v WARNING reduce los mensajes que muestra la consola mientras renderiza. Para ver la lista completa de opciones, ejecutá manim -h en tu terminal.
07Tips para tus primeras animaciones
El nombre de la clase importa
El comando de consola debe usar exactamente el mismo nombre que pusiste después de class. Si no coinciden, Manim no encuentra qué renderizar.
wait() no es opcional para ver algo
Si encadenás varios play() sin wait(), las animaciones pasan tan rápido que parecen no haber ocurrido. Usalo generosamente al principio.
Empezá en baja calidad
Usa siempre -ql mientras estás probando y ajustando. Subí a -qh (alta calidad) solo en la versión final — renderizar en alta calidad es mucho más lento.
Cada objeto puede transformarse en otro
Transform() funciona entre casi cualquier par de objetos (texto, formas, gráficos) — es la base de gran parte de la magia visual de Manim.
